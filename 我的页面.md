# 「我的」页面设计说明

> 目标：在 Flutter 侧还原 daily-coach-ai 中 Profile 页的体验，聚焦个人信息展示、目标与偏好入口、应用版本等内容，保持多彩、柔和的视觉风格。

---

## 1. 页面定位与入口
- **导航位置**：底部 Tab「我的」，对应 GoRouter 的 `/app/profile`。
- **用户状态**：需已登录且完成引导；页面加载时拉取用户基本信息、目标状态以及设置项（可先使用 mock）。
- **核心使命**：提供个人信息展示、目标配置入口、偏好设置、隐私/帮助等二级页面入口，并展示应用版本。

---

## 2. 信息结构
1. **顶部标题**
   - 文案：「个人中心 👤」或根据用户昵称动态调整。
   - 放置在页面起始位置，配合简短的欢迎语（可选）。
2. **个人卡片**
   - 内容：头像/emoji、昵称、激励文案（例如「改变自己的第 28 天」）。
   - 形式：渐变背景 + 3D 阴影 + 边框，整体呈现胶囊感。
3. **模块分组**
   - 「我的目标」
     - 健康目标设置
     - 工作&学习目标
     - 阅读清单
   - 「偏好设置」
     - 通知提醒
     - AI 风格设置
   - 「隐私与支持」（可在 Flutter 版拓展）
     - 隐私与数据（导出/删除）
     - 帮助与反馈
     - 关于/版本号
4. **应用版本信息**
   - 页面底部居中展示，例如「v1.0.0 日常教练」。

---

## 3. UI & 交互要求
- **背景**：延续全局 mesh 背景 + 半透明内容容器，页面主体使用浅灰（`surface`）作为基色，卡片为白底。
- **卡片样式**：
  - 圆角 24–32px。
  - `border: 1px solid #F1F5F9`，阴影可参考 `shadow-card`。
  - 头像区域可使用 `CircleAvatar` 或 emoji 容器，带渐变背景及白色描边。
- **菜单项（MenuLink）**：
  - 行高 56–64px，左右布局，左侧放 icon + label，右侧放箭头 icon。
  - 点击态：背景轻微变浅，icon/文字颜色高亮（如糖果紫）。
  - 列表整体使用 `ListView` 或 `Column`，通过 `Container`/`Card` 包裹，首尾行圆角处理。
- **滚动**：
  - 页面整体使用 `CustomScrollView` / `SingleChildScrollView`，底部需预留 80+px 以避开底部导航。
- **交互**：
  - 点击菜单项跳转对应设置页面（可先 `TODO` 路由）。
  - 头像卡片可支持点击进入个人资料编辑。

---

## 4. 数据与状态
- **个人信息**：昵称、emoji/头像、连续天数等来自用户资料，可放入 `ProfileState`（Riverpod）。
- **菜单配置**：推荐以常量列表驱动，包含 icon、文案、目标路由/回调。
- **版本号**：从 `package_info_plus` 获取，初期可写死。

## 6. Flutter 实现要点
1. **布局**：使用 `CustomScrollView` + `SliverToBoxAdapter` 组合，便于适配不同屏幕。
2. **组件复用**：抽象 `ProfileMenuCard`、`MenuLinkTile`、`ProfileHeroCard` 等 Widget。
3. **动画/手势**：头像卡片加轻微 scale 动效；菜单项使用 `InkWell`/`GestureDetector`，带亮色过渡。
4. **状态管理**：预留 `profileController`（StateNotifier/Riverpod），以便后续接入真实数据。
